7. Per-OS & Device Layouts

7.1 Linux Desktop (Wide, ≥600px)

⚙NetDiagnostic
Target input + Layer checkboxes + Port scan + Summary sidebar

Results list + progress panel

7.2 Windows Desktop (Wide, ≥600px)

Same layout as Linux Desktop. Windows-specific: title bar matches system theme. App uses Win32 native diagnostic APIs via nd_plugin.

7.3 macOS Desktop (Wide, ≥600px)

Same layout as Linux Desktop. macOS-specific: native title bar integration. App uses CoreWLAN / IOKit / SystemConfiguration for diagnostics.

7.4 Android Phone (Narrow, 360–448 dp wide × 780–1008 dp tall)

Supported Android device references: Pixel 8/9 series (412×915 → 448×1008 dp), Galaxy S24 series (360×780 → 384×854 dp), OnePlus 12 (412×917 dp). Compact nav bar (icon-only, right-aligned, 32px height). Sidebar scrolls above content in narrow layout. TargetInput and SummaryCards remain visible in the compact sidebar section.

7.5 iOS iPhone (Narrow, 375–440 dp wide × 667–956 dp tall)

Supported iOS device references: iPhone SE 3rd gen (375×667 dp, home button), iPhone 15 Pro (393×852 dp, Dynamic Island), iPhone 15 Pro Max (430×932 dp, Dynamic Island), iPhone 16 (402×874 dp, Dynamic Island), iPhone 16 Pro Max (440×956 dp, Dynamic Island).

Same narrow layout as Android. Key differences: iOS sandbox limits G2/G3/G4 diagnostics (returns SKIPPED with degradation note). Safe area padding for notch/Dynamic Island (44pt top inset). Compact nav bar (32px). Devices with Dynamic Island (iPhone 14 Pro+ and iPhone 15+ series) render an island indicator.

7.6 iOS iPad / Android Tablet (Wide, ≥600dp reference)

Supported tablet devices: iPad mini 6th gen (744×1133 dp), iPad Pro 11″ M4 (834×1210 dp), iPad Pro 12.9″ M4 (1024×1366 dp). Tablets trigger the Wide layout at ≥600dp, with the full 260px sidebar visible. Landscape tablets get the sidebar + content side-by-side. Portrait tablets with width <600dp fall back to narrow stacked layout.
